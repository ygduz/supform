# Supform Mobile (Capacitor)

iOS and Android native shell for Supform. Wraps the existing web app with no changes
to the web source. The `mobile/` folder is entirely self-contained — the web app at
`frontend/` and `backend/` are untouched.

## Architecture

```
Supform repo
├── backend/          unchanged — runs on your server
├── frontend/         unchanged — same React source
└── mobile/           ← you are here
    ├── capacitor.config.ts    Capacitor configuration
    ├── package.json           mobile-only deps
    ├── ios/                   generated by `cap add ios` (macOS only)
    └── android/               generated by `cap add android`
```

Capacitor bundles `frontend/dist/` into a native WebView. Its built-in local server
rewrites unmatched paths to `index.html`, so React Router's `BrowserRouter` works without
any code change.

## Step-by-step (first time, on macOS)

### 1. Configure your backend URL

```bash
cd mobile
cp .env.example .env
# Edit .env — set VITE_API_BASE_URL to your deployed HTTPS backend:
# VITE_API_BASE_URL=https://api.yourdomain.com
```

The mobile app cannot talk to `localhost:8000`. You need a deployed backend first
(see "Deploying the backend" below).

### 2. Install mobile dependencies

```bash
cd mobile
npm install
```

### 3. Build the web bundle

```bash
npm run build:web
# This runs `vite build` in ../frontend with your VITE_API_BASE_URL injected.
# Output goes to ../frontend/dist/ which capacitor.config.ts points at.
```

### 4. Add native platforms (first time only)

```bash
npx cap add ios
npx cap add android
```

This generates `mobile/ios/` and `mobile/android/` directories with the native projects.

### 5. Add iOS permission strings

Open `mobile/ios/App/App/Info.plist` and add inside the root `<dict>`:

```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>Used to collect location answers on forms.</string>

<key>NSCameraUsageDescription</key>
<string>Used to scan barcodes and capture photos on forms.</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>Used to attach photos to form responses.</string>
```

These are required because the renderer uses geolocation (geopoint fields), the camera
(barcode scanner), and the photo library (image upload fields). Apple will reject the app
without them.

### 6. Sync web assets to native projects

```bash
npm run sync
# Equivalent to: npx cap sync
# Copies ../frontend/dist/ into ios/ and android/ and installs native plugins.
```

### 7. Open in Xcode / Android Studio

```bash
npm run open:ios      # opens Xcode
npm run open:android  # opens Android Studio
```

In Xcode:
- Select your Apple Developer team under **Signing & Capabilities**
- Choose a simulator or connected device
- Press ▶ to run

### 8. Test on device / simulator

Use `npm run run:ios` or `npm run run:android` to launch directly from the terminal once
signing is configured.

### 9. Archive and submit to App Store

In Xcode:
1. Select **Any iOS Device** as the target
2. **Product → Archive**
3. In the Organizer, click **Distribute App → App Store Connect**
4. Follow the upload wizard
5. In [App Store Connect](https://appstoreconnect.apple.com), complete the listing
   (screenshots, description, privacy policy URL, age rating) and submit for review

## Backend CORS configuration

Capacitor serves from `capacitor://localhost` (iOS) and `https://localhost` (Android).
Add these to your deployed backend's `SUPFORM_CORS_ORIGINS` environment variable:

```
SUPFORM_CORS_ORIGINS=["https://yourdomain.com","capacitor://localhost","https://localhost"]
```

No backend code changes needed — it reads this from env already.

## Deploying the backend

The mobile app needs an HTTPS backend. Recommended setup:

| Service | What runs there | Cost |
|---|---|---|
| [Railway](https://railway.app) | backend + worker + Postgres + Redis | ~$5-10/mo |
| [Vercel](https://vercel.com) | frontend web (optional browser version) | free |

Railway can deploy directly from `docker-compose.yml`. Steps:
1. Push the repo to GitHub
2. New Railway project → Deploy from GitHub
3. Set env vars: `SUPFORM_SECRET_KEY`, `SUPFORM_DATABASE_URL`, `SUPFORM_REDIS_URL`,
   `SUPFORM_CORS_ORIGINS` (include Capacitor origins above)
4. Run migrations: `railway run alembic upgrade head`

## Updating the app

When the web app changes:

```bash
npm run build:web   # rebuild
npm run sync        # push to native
# Open Xcode → Archive → upload new build
```

Capacitor plugins only need to be updated when you change `mobile/package.json` deps
and re-run `npm install && cap sync`.
